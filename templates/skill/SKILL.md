# {{title}}

## 适用场景

当用户需要……时使用这个 skill。

不要在以下场景使用：

- 用户目标不属于本 skill 的业务域。
- 用户只是在泛泛提问，不需要专门工作流。
- 需要联网、写文件、调用外部系统，但用户尚未授权或上下文不足。

## 输入条件

执行前确认这些信息：

1. 用户目标。
2. 输入文件、仓库、页面、数据或系统范围。
3. 输出形式，例如代码修改、报告、文档、截图或结论。
4. 验收标准。

## 工作流程

1. 读取本文件，确认是否适用。
2. 只读取当前任务需要的 `references/` 文件。
3. 检查本地文件和上下文，避免基于假设执行。
4. 执行最小必要改动或分析。
5. 运行可用验证命令。
6. 用清晰结论回复用户，明确已完成事项、验证结果和未覆盖边界。

## 引用资料

- `references/README.md`：记录本 skill 的资料路由和维护说明。
- `scripts/README.md`：记录可执行脚本的用途和运行方式。
- `assets/README.md`：记录可复用资产。
- `tests/README.md`：记录测试样例、夹具和验收方式。
