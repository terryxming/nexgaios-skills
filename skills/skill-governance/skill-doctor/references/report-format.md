用本文件生成稳定、中文化、可执行的诊断报告。

```markdown

一句话判断：...

## Top Issues

| 优先级 | 视角 | 问题 | 证据 | 后果 | 修复建议 |
|---|---|---|---|---|---|
| P0 | 生产端 | ... | `SKILL.md` ... | ... | ... |

## 三视角诊断

### 资深 AI 产品经理视角
- ...

### 生产端视角
- ...

### 消费端视角
- ...

## 触发与误触发测试

| 类型 | Prompt | 预期 | 当前风险 |
|---|---|---|---|

## 建议改造路径

1. 最小修复：...
2. 推荐修复：...
3. 后续增强：...

## 验证与未覆盖边界

- 未验证：...
```

## Release Mode 报告

```markdown
## 发布门禁结论

结论：通过 / 暂缓 / 阻塞
目标版本：vX.Y.Z
当前状态：draft / candidate / released / deprecated / unknown

## 阻塞项

| 优先级 | 问题 | 证据 | 风险 | 修复建议 |
|---|---|---|---|---|

## 版本与生命周期检查

- 已发布版本冻结：通过 / 失败 / 未知
- CHANGELOG 倒叙：通过 / 失败 / 未知
- 当前版本可追溯：通过 / 失败 / 未知
- 回滚目标明确：通过 / 失败 / 未知
- 测试记录完整：通过 / 失败 / 未知
- 复盘信息完整：通过 / 失败 / 未知

## 变更摘要

### 新增
- ...

### 变更
- ...

### 修复
- ...

### 移除
- ...

## 发布建议

1. ...
```

## 写作规则

- 先给结论，再给证据。
- P0/P1/P2 分开，不把偏好包装成缺陷。
- 每条问题写清用户后果，避免只写内部实现问题。
- 修改建议优先给最小修复；重构建议放后面。
- 不要输出大段泛泛原则；把原则落实到目标 skill 的具体位置。
