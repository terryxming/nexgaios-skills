# Changelog

本文件记录 ob-notes 对外版本间的变化（面向使用者）。开发细节与决策记录见 skill 内的 dev-log（面向维护者）。

版本遵循语义化版本 `MAJOR.MINOR.PATCH`，规则见 `references/maintenance.md` 第 5 节。

## [0.7.0] - 2026-06-27

source-fidelity 从"仪式闸"升级为"实质闸"：覆盖表不再能当遮羞布（MINOR，向后兼容）。

- **覆盖表锁粒度**：默认保留类章节枚举到最细层级（有 H3 到 H3）；N 个子节折一行 = 省略 N-1 节、须逐节点名丢失内容；"拆成表/理论压掉"不再算省略理由。
- **放行改用实质闸**：以"被压缩的机制能否只凭笔记读懂、不回原文"取代"落盘 <1/4 就补覆盖表"这一可被一张表满足的旧出口；字数比率降级为加重审查的触发器。
- **保留清单补"机制怎么运作 + 确切模块名/工具名/参数"**（呼应铁律二），并加**源上限护栏**：只补原文确有的，原文没展开的深层不许编（铁律三）、改记为待深挖钩子。
- 触发：dogfood 复盘发现 AWS 记忆文章笔记虽出了覆盖表，却把各框架 H3 机制（Mem0 六层模块名、Letta 工具名等）压没——覆盖表停在 H2 粒度、旧 1/4 出口被一张表满足。

## [0.6.0] - 2026-06-26

迭代审计的根因治理：把 SSOT 从"规则定义"延伸到"派生事实"，消除散文里的拷贝（MINOR，向后兼容）。

- **去重**：归属表 / SKILL 引用清单里对坏例名单、各类计数（五条铁律 / 三档 / 五步…）的手抄，改为引用唯一家或去掉计数。顺带修一处已漂的坏例名单（归属表误写"入口硬"、漏"过度保留"）。
- **quality-check 的 30 秒阅读测试**改为测"产出"（读者 30 秒内拿到回报），字段规格归 mode-a-research、不再复列。
- **maintenance 立原则**：SSOT 也管派生事实——可派生别手写、必须就地用引用、消不掉的在改约定时做提及扫描。
- **§6 修改流程加"提及扫描"**：改规则时除跑脚本查结构依赖外，手动 grep 其名字 ＋ 派生事实 across 全库，堵脚本看不见的散文漂移。
- 评估后**未采用**配置式哨兵 lint：范式化已消除主要拷贝，lint 属看护残渣 ＋ 有"改时记得登记"的同源漏洞，过度工程；留待范式化证明不足再议。

## [0.5.1] - 2026-06-26

迭代审计后的口径校正（PATCH，无行为变化）。

- 修"四测"旧计数：quality-rubric 自 v0.4 起已是五测（含研究型掌握测试），同步 quality-check 标题、SKILL 引用清单、maintenance 归属表。
- 命名 / 双链 / 监控示例由英文 slug 改为标题式，对齐 v0.5 的"文件名 = 显示标题"（mode-a-practice、frontmatter-tags、monitoring）。
- anti-patterns 第一屏示例字段对齐新 30 秒读法模型（是什么 / 解决什么 / 最重要结论）。
- 30 秒读法「边界」与正文「适用边界」消歧：前者 = 能力速览，后者 = 适用范围，互指。
- SKILL 引用清单补全 layout-rule / mastery-lens / source-fidelity 的提及。

## [0.5.0] - 2026-06-26

**破坏性变更**（改了命名约定与模板结构；已有 slug 命名的笔记不会失效，但与新约定不一致，可逐步对齐）。

- **文件名 = 显示标题**（naming-rule 改向）：不再用英文 slug，文件名直接用显示标题（仅规避文件系统非法字符）。理由：落点固定是 Obsidian 库、改名自动修链，slug 的跨工具收益基本不成立，反而徒增 slug ↔ 标题的心智映射。dev-log 固定名为例外。
- **研究 / 实战模板去掉 H1**：文件名即标题、靠 Obsidian"显示行内标题"承载，正文不再写 H1，消除双标题。dev-log 保留 H1（文件名固定不表标题）。
- **30 秒读法字段升级**（研究型）：由"这是什么 / 为什么值得留 / 最重要结论 / 可信边界"改为 **是什么 / 解决什么 / 最重要结论 / 怎么用 / 前置 / 边界**——从归档视角转向学习与使用视角；"前置"= 要看懂得先会什么，"边界"= 能力边界（非可信度）。
- **frontmatter-spec** 写清"文件名 / 行内标题 / frontmatter title"三者关系。
- 配套（库级、不属本 skill）：Obsidian 极简阅读型 CSS snippet，统一观感。

## [0.4.0] - 2026-06-26

围绕"让笔记驱动掌握、把版式钉在克制上"的新增，均向后兼容（不改模板结构、不动已有笔记）。

- **掌握视角（mastery-lens，研究型）**：研究型笔记新增"学习闭环"写作纪律——落笔前自问"能讲清 / 何时不成立 / 能迁到哪 / 连得上谁"，把掌握织进行文；并明确这是作者内功，**不得做成笔记章节**（防伪掌握、防方法论上台面）。
- **排版规约（layout-rule）**：新增 markdown 元素的用途与克制规约（每元素一职责、段落优先、callout 克制、可视化按需），并划清"内容层格式（本 skill 管）"与"呈现层样式（Obsidian 主题 / CSS，库级）"。
- **质量自检加「掌握测试」**：quality-rubric 增第 5 测，检验研究型笔记"读起来像真掌握，还是只复述了资料"。

## [0.3.0] - 2026-06-26

从前作 obsidian-knowledge-curator 捞取五项"编辑智慧"并入（只移植内容质量经验，不移植预览-确认闭环、母文档编号体系、视觉/控制台等流程重量）。新增 3 个规则项、2 个 reference，均向后兼容。

- **源材料保真（source-fidelity，研究型）**：沉淀网页/长文/报告时，正文开头放来源元数据块（链接/标题/作者/发布日期/笔记日期），并给一张原文结构覆盖表（每个主章节标 保留/压缩/拆分/省略，省略给理由）——防止长文被压成观点卡。
- **反例库（anti-patterns）**：新增 `references/anti-patterns.md`，给六类坏笔记（流水账/空泛/过度保留/过度提炼/格式炫技/Agent-only）的坏例与改写。
- **写盘前质量自检（quality-rubric）**：新增 `references/quality-check.md`，落盘前过四测（30秒阅读/信号噪音/证据/复用），与两周复盘分工互补。
- **模板增强**：研究型/实战型骨架加"适用边界 / 下次怎么用"段，第一屏摘要升级为 30 秒读法契约；双链占位提示按 linking-convention 标 (待建)。

## [0.2.0] - 2026-06-26

真实沉淀演练暴露的三处精修，均向后兼容（已有笔记不受影响）。

- **可信度判据收紧**：明确"已验证"指**内容已亲验**（亲手跑通 / 复现 / 核对一手来源），而非"信源可信"。读到可信文章但未亲验的结论默认标"待验证"，拿不准就低估。
- **新增可选 `source_url` 字段**：笔记 frontmatter 可记一手原文链接，便于日后溯源与升档前回查；研究型 / 实战型模板骨架已纳入。
- **双链防死链**：建 `[[双链]]` 前先查目标笔记是否存在；想预留尚未建的链时用 `(待建)` 显式标注，不留隐形死链。

## [0.1.0] - 2026-06-25

首个版本（早期，尚未承诺稳定 API）。

- 双 Mode 沉淀：知识沉淀（Mode A：研究型 / 实战型）回写 Obsidian，项目记忆（Mode B：dev-log）跟项目走。
- 五条铁律：preflight 校验、保留确切细节、可信度三档、追加不覆盖、项目先读 dev-log。
- 可配置知识库根路径 `{kb_root}`，配置存于 skill 之外。
- 中英双轨 tag：笔记存中文、英文作对接别名。
- 监控与复盘：jsonl 沉淀日志 + 回访信号，两周复盘（回访次数不作删除依据）。
- 维护治理层：单一真相源 + 依赖声明，`build_depmap.py` 自动校验 MECE 并生成依赖图。
