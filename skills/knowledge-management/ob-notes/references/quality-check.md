# 写盘前质量自检 — 四测

**这是写盘前对单篇笔记的最后一道闸**：不合格先重写，别把问题留给未来的自己。它与 monitoring 的"两周复盘"分工不同——复盘是事后看全库哪些被用上，本表是此刻看这一篇够不够好。

抽取完、套好模板、写盘前，逐项过一遍：

## 1. 30 秒阅读测试（治"没重点"）

第一屏（summary callout）必须让人在 30 秒内拿到：**这是什么 / 为什么值得留 / 最重要结论 / 以后何时用 / 可信或适用边界**。

失败信号：开头先解释文档怎么读、编号体系或内部流程；读完第一屏还不知道收获是什么。

## 2. 信号噪音测试（治"压缩过狠/不足"）

留下决策、根因、证据、边界、可复用动作；压掉临时尝试、重复命令、聊天铺垫、工具日志。

失败信号："做了什么"多于"学到了什么"；每个步骤权重相同、没有主次。命中即对照 `anti-patterns.md` 重写。

## 3. 证据测试（治"伪知识"）

重要结论能回到来源、观察、用户原话或文件；没有亲验的不写成确定结论——按 credibility-spec 标 `[待验证]`，必要时放进"适用边界 / 待验证"。外部事实无一手出处时不写成事实（配合 frontmatter 的 source_url）。

失败信号：只有结论没有依据；只有原则没有例子。

## 4. 复用测试（治"只能回忆、不能指导"）

明确"**下次遇到什么情况**用这篇"，给出检查清单、判断准则或可迁移模式，让未来的你或 agent 少走一步弯路。

失败信号：看完不知道如何行动；只能回忆历史。

## 最低合格线

一篇正式落盘至少满足：30 秒阅读测试过、信号噪音测试过、证据测试过；若材料是网页/长文/报告，还要过 `mode-a-research.md` 的 source-fidelity（原文结构覆盖）。任一不过，先重写再写盘。
