# 参考资料

本目录存放 `obsidian-knowledge-curator` 的渐进式披露资料。不要一次性全部读取；按任务需要选择。

资料路由：

- `information-types.md`：需要判断笔记类型、选择组织结构或拆分多篇笔记时读取。
- `anti-patterns.md`：需要审查旧笔记、识别噪音、设计反例或改写坏片段时读取。
- `markdown-layout-patterns.md`：需要设计 Obsidian / Markdown 版式、frontmatter、callout、表格、wikilink 或标签时读取。
- `obsidian-visual-patterns.md`：需要制作 Obsidian 美化版、CSS snippet、callout 样式、表格样式、Mermaid 流程图或视觉渲染验收时读取。
- `quality-rubric.md`：需要做输出验收、30 秒阅读测试、误触发测试或 forward-test 时读取。
- `evaluation.md`：需要评估网页/长文沉淀的原文覆盖、信息保真、压缩过度和业务能力时读取。
- `lifecycle-policy.md`：需要处理冻结编号、预览版本流转、落盘替代、废弃版本、skill 版本升级或 skill 源码维护闭环时读取。
- `project-memory.md`：需要维护 skill 项目母文档、项目上下文、项目架构、迭代时间轴、决策索引、未闭环事项，或在修改 skill 源码、规则、reference、模板、样本、测试、版本、安装状态后检查母文档同步影响时读取。

维护要求：

- 文件名使用小写 kebab-case。
- 每个引用文件只服务一个明确主题。
- 反例必须脱敏、泛化，不提交用户真实旧笔记原文。
- 修改 reference 或维护规则时，必须同步检查根目录 `impact.yaml` 的契约组是否仍覆盖相关文件和 witness。
